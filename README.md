# Konechniavtomat
# ДКА: Процесс двухфакторной аутентификации (2FA)
# Тип: Детерминированный конечный автомат (ДКА).
# Суть: Пользователь должен пройти строгую цепочку шагов для входа в систему.

    Определение компонентов автомата
    Множество состояний Q:
    
    q0 — Ожидание ввода логина (начальное состояние).
    
    q_pwd — Логин принят, ожидание пароля.
    
    q_2fa — Пароль верен, ожидание кода из SMS/приложения.
    
    q_success — Вход выполнен (конечное/допускающее состояние).
    
    q_blocked — Аккаунт временно заблокирован из-за ошибок (поглощающее состояние ошибки).
    
    q_err — Общая ошибка аутентификации (поглощающее состояние).
    
    Входной алфавит Σ = {L, P, C, X, R}:
    
    L — Ввод корректного логина (Login).
    
    P — Ввод верного пароля (Password).
    
    C — Ввод верного кода 2FA (Code).
    
    X — Ввод неверных данных (логина, пароля или кода).
    
    R — Сброс/начало заново (Reset). Разрешен только из финальных состояний, в реальном коде — переход на новую сессию.
    
    Начальное состояние: q0.
    
    Конечные (допускающие) состояния: F = {q_success}.
    
    Поглощающие состояния: q_blocked, q_err.

    q0 — старт (кружок с входящей стрелкой).
    
    q_success — конечное состояние (двойной кружок или рамка).
    
    q_blocked / q_err — поглощающие состояния (стрелки зациклены на себя или явно указаны как ловушки без выхода по рабочим символам).
    
    Проверка работы автомата
    1. Успешный вход: L, P, C
    
    q0 → (L) → q_pwd
    
    q_pwd → (P) → q_2fa
    
    q_2fa → (C) → q_success
    
    Результат: ДОПУСК (Вход разрешен).
    
    2. Ошибка на этапе логина: P, ...
    
    q0 → (P) → q_err
    
    Результат: ОТКЛОНЕНА (Нарушение процесса).
    
    3. Блокировка после ошибки пароля: L, X
    
    q0 → (L) → q_pwd
    
    q_pwd → (X) → q_blocked
    
    Результат: ОТКЛОНЕНА (Аккаунт заблокирован).
    
    4. Пропущен шаг 2FA: L, P, L
    
    q0 → (L) → q_pwd
    
    q_pwd → (P) → q_2fa
    
    q_2fa → (L) → q_err (Нельзя вводить логин, когда ждут код)
    
    Результат: ОТКЛОНЕНА (Нарушение процесса).
    
    5. Сброс после успеха: L, P, C, R, L, P, C
    
    Первая тройка: q0 → ... → q_success (ДОПУСК).
    
    Далее R: q_success → q0 (Начинаем новую сессию).
    
    Вторая тройка: q0 → ... → q_success (ДОПУСК).
    
    Результат: ДОПУСК (Разрешено начать заново).
